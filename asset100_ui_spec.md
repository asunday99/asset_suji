# ASSET 333 - UI 재작성 명세서 (for 안티그래비티)

본 문서는 ASSET 333 앱의 UI를 "The Baseline" 스타일(딥블랙, 카드 코너 마크, 배경 패턴, 프리미엄 폰트 등)로 전면 재작성하기 위한 가이드입니다.
로직과 데이터 연동 부분은 이미 분리되어 있으므로, 명세된 변수들을 활용하여 화면(UI) 렌더링에 집중해 주시면 됩니다.

## 1. 전역 디자인 요구사항
* **프레임워크**: Streamlit (Python)
* **배경색**: 딥블랙 계열 (완전 블랙 또는 매우 어두운 네이비)
* **폰트**: 프리미엄 금융 앱 느낌을 주는 폰트 (예: Pretendard 등 가독성 높고 세련된 폰트)
* **카드 스타일**: Glassmorphism 또는 뚜렷한 테두리, 카드 코너 마크 등 고급스러운 디테일 추가
* **포인트 컬러**: 주황색은 꼭 필요한 곳(강조, 긍정적 수익 등)에만 최소한으로 사용하고, 나머지 차트/그래프는 소프트블루(`#5B8DEF`), 연보라(`#9B72CF`) 등 세련된 톤 사용
* **반응형**: 대시보드와 매매기록 탭 모두 `min(1200px, 90vw)` 의 최대 폭을 가지며, 모바일에서는 1열, PC에서는 적절히 배치되도록 구현

## 2. 탭 구성
`streamlit_option_menu`를 사용하여 메인 탭 2개를 구성합니다.
* **대시보드**
* **매매 기록**

## 3. 대시보드 탭 명세

대시보드는 **완전 1열 반응형 레이아웃**으로 구성됩니다.

### 3.1 목표 달성 및 총 자산 (최상단)
* **입력 변수**:
  * `gs_val`: 목표 금액 (억 단위, 예: 50)
  * `d_days_dynamic`: 목표일까지 남은 일수
  * `total_assets`: 현재 총 자산 (원 단위)
  * `ach`: 목표 달성률 (%)
* **요구 UI**:
  * 상단에 "목표 {gs_val}억 | D-{d_days_dynamic}" 텍스트 표시 (가독성 높은 회색/흰색 조합)
  * 그 아래 매우 큰 네온/강조 폰트로 총 자산 `₩{total_assets:,}` 표시
  * 달성률을 나타내는 프로그레스 바 (주황/골드 그라데이션) 및 하단에 `{ach:.2f}% 달성` 텍스트 표시

### 3.2 목표 재설정 (Expander)
* **상태**: 기본 닫힘 (`expanded=False`)
* **입력 요소**:
  * `st.number_input`: 목표 금액 입력 (원 단위)
  * `st.date_input`: 목표일 선택
* **동작**: 값 변경 시 `st.session_state` 업데이트 및 저장 로직 호출 (기존 코드 참조)

### 3.3 매크로 지표 (Expander)
* **상태**: 기본 닫힘 (`expanded=False`)
* **입력 변수**: `mapped_vals` (지표명 -> 값 딕셔너리), `macro_changes` (지표명 -> 변화량 딕셔너리)
* **요구 UI**:
  * 10개의 핵심 매크로 지표를 모바일 2열, PC 4열 형태의 그리드로 표시
  * 각 지표 카드는 얇고 세련된 테두리를 가지며, 지표명(작게), 현재값(크게), 변화량(색상 적용)을 표시

### 3.4 자산 카드 (가로 스크롤)
* **입력 변수**: 주식, 금, 채권, 코인, 현금 등 각 자산별 현재가, 비중, 손익 등의 데이터 딕셔너리 리스트
* **요구 UI**:
  * 가로 스크롤이 가능한 `swipe-container` 구현
  * 스크롤 가능함을 알리는 chevron(삼각형) 표시
  * 각 카드는 해당 자산의 테마 색상(주식: 파랑, 금: 노랑, 코인: 주황 등)을 은은하게 반영한 Glassmorphism 스타일

### 3.5 차트 영역 (SVG Area 차트 좌우 병렬)
* **입력 변수**:
  * 차트 1: 월별 총자산 추이 (`_monthly_last_asset`)
  * 차트 2: 월별 목표 달성률 추이 (`_monthly_ach`)
* **요구 UI**:
  * 좌우로 나란히 배치 (PC 기준, 모바일은 상하 배치)
  * 세련된 SVG Area 차트로 렌더링 (차트 1은 소프트블루, 차트 2는 연보라 테마)

### 3.6 포트폴리오 비중 (도넛 차트)
* **입력 변수**: 자산별 비중 데이터
* **요구 UI**: Plotly를 이용한 도넛 또는 Sunburst 차트. 색상은 눈부시지 않은 세련된 톤으로 커스텀.

### 3.7 현황 요약 보기 (Expander)
* **상태**: 기본 닫힘 (`expanded=False`)
* **요구 UI**: 상세 테이블 데이터(`df_dash`)를 보기 좋게 포맷팅하여 표시

## 4. 매매 기록 탭 명세

### 4.1 이번 달 수익 요약 (최상단)
* **입력 변수**: `_month_profit` (이번 달 실현 손익)
* **요구 UI**:
  * 눈에 띄는 큰 배너 형태로 "이번 달 팔아서 +O,OOO원 벌고 있어요!" (또는 손실/0원 메시지) 표시
  * 수익은 적색/주황색, 손실은 청색 계열 적용

### 4.2 매매 캘린더
* **입력 변수**: `daily_pnl_dict` (날짜 -> 손익 딕셔너리), `_disp_year`, `_disp_month`
* **요구 UI**:
  * 상단에 `◀` `▶` 버튼으로 월 이동 가능하게 구현 (session_state 연동)
  * 앱 로드 시 항상 '오늘 달'이 표시되어야 함
  * 하한선: 2026년 1월 (이전으로 이동 불가)
  * 상한선: 오늘 달 (이후로 이동 불가)
  * 달력 테이블 내에 각 일자별 실현 손익을 표시 (양수/음수 색상 구분)

### 4.3 버블 차트 영역 (좌우 병렬)
* **입력 변수**:
  * 차트 1: 월별 실현 손익 (`_monthly_profits_c`, `_monthly_counts_c2`)
  * 차트 2: 월별 수익률 (`_monthly_avg_rates2`, `_profit_as_count`)
* **요구 UI**:
  * 대시보드 차트와 마찬가지로 좌우 나란히 배치
  * SVG 기반의 버블 차트로 구현 (버블 크기와 색상으로 데이터 표현)

### 4.4 매매 기록 테이블 및 모의 계산
* **요구 UI**:
  * 실제 매매 기록 테이블 (`df_real`) 표시
  * 그 아래에 모의 계산 테이블 (`df_mock`)을 `st.data_editor`로 제공하여 사용자가 입력/수정 가능하도록 구현

### 4.5 익절/손절 계산기
* **요구 UI**:
  * 종목명/티커 입력 필드
  * 전일 종가 기반으로 설정된 수익률/하락률에 따른 목표가/손절가 테이블 (`st.data_editor`) 표시

## 5. 주의 사항
1. **로직 함수 분리**: 기존 `app.py`에 있던 복잡한 데이터 로드, 계산, 정제 함수들은 모두 `logic.py`로 분리되었습니다. UI 코드에서는 `from logic import *`를 통해 이 함수들을 호출하여 데이터만 받아와야 합니다. **절대 UI 파일 안에서 데이터를 재계산하거나 로직을 수정하지 마세요.**
2. **달력 로직 유지**: 달력의 월 이동은 `_trade_cal_year`, `_trade_cal_month`, `_trade_cal_navigated` session_state 키를 사용하는 A방식 로직을 반드시 유지해야 합니다.
3. **날짜 파싱 주의**: 데이터 로드 시 날짜 컬럼을 숫자로 변환하지 않도록 처리된 로직을 존중해야 합니다.
4. **Streamlit 제약**: 완벽한 픽셀 매칭보다는 Streamlit의 한계 내에서 최대한 고급스러운 느낌을 내는 CSS/HTML 주입을 권장합니다.
